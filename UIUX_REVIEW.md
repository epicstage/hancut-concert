## ⚠️ 개선이 필요한 부분

### 1. 시인성 개선 사항

#### 🔴 심각 (즉시 개선 권장)

**1.1 글래스모피즘 배경 가독성**
- **문제**: 일부 텍스트가 반투명 배경에서 가독성이 떨어질 수 있음
- **위치**: `glass-card` 내부의 작은 텍스트
- **영향**: 밝은 배경에서 텍스트가 흐릿하게 보일 수 있음
- **개선 방안**:
  ```css
  /* 텍스트 가독성 향상 */
  .glass-card p, .glass-card span {
    text-shadow: 0 1px 2px rgba(0, 0, 0, 0.1);
  }
  ```

**1.2 에러 메시지 색상 대비**
- **문제**: 에러 메시지가 작은 폰트(`0.875rem`)로 표시됨
- **위치**: `input-error-message` 클래스
- **영향**: 시력이 약한 사용자가 에러 메시지를 놓칠 수 있음
- **개선 방안**:
  ```css
  .input-error-message {
    font-size: 0.9375rem; /* 15px로 증가 */
    font-weight: 600; /* 굵게 */
  }
  ```

#### 🟡 중요 (개선 권장)

**1.3 힌트 텍스트 가독성**
- **문제**: `input-hint`가 `text-muted` 색상으로 너무 연함
- **위치**: 입력 필드 하단 힌트
- **개선 방안**:
  ```css
  .input-hint {
    color: var(--text-secondary); /* 더 진한 색상 */
    font-size: 0.875rem;
  }
  ```

**1.4 배지 텍스트 크기**
- **문제**: 배지 내부 텍스트가 작음 (`0.875rem`)
- **위치**: `badge` 컴포넌트
- **개선 방안**:
  ```css
  .badge {
    font-size: 0.9375rem; /* 15px */
    padding: 0.5rem 0.75rem; /* 패딩 증가 */
  }
  ```

### 2. 사용성 개선 사항

#### 🔴 심각 (즉시 개선 권장)

**2.1 키보드 네비게이션**
- **문제**: 일부 버튼에 `tabindex` 설정이 없음
- **위치**: 아이콘 버튼, 모달 닫기 버튼
- **영향**: 키보드만 사용하는 사용자가 접근 불가
- **개선 방안**:
  ```html
  <button class="btn-icon" tabindex="0" aria-label="닫기">
  ```

**2.2 폼 제출 에러 처리**
- **문제**: 네트워크 에러 시 사용자 피드백이 부족
- **위치**: `apply.html`, `check.html`의 폼 제출
- **개선 방안**: 
  - 로딩 상태 명확히 표시
  - 타임아웃 에러 처리 추가
  - 재시도 버튼 제공

**2.3 모바일에서 테이블 가로 스크롤**
- **문제**: 관리자 페이지 테이블이 모바일에서 잘림
- **위치**: `admin.html`의 참가자 테이블
- **개선 방안**:
  ```css
  .data-table {
    min-width: 800px; /* 최소 너비 설정 */
  }
  .glass-card {
    overflow-x: auto; /* 가로 스크롤 활성화 */
  }
  ```

#### 🟡 중요 (개선 권장)

**2.4 입력 필드 자동완성**
- **문제**: 일부 필드에 `autocomplete` 속성 누락
- **위치**: 주민번호 앞자리 입력 필드
- **개선 방안**:
  ```html
  <input autocomplete="bday" />
  ```

**2.5 로딩 상태 표시**
- **문제**: 일부 비동기 작업에서 로딩 인디케이터가 없음
- **위치**: 관리자 페이지 데이터 새로고침
- **개선 방안**: 스켈레톤 로딩 또는 스피너 추가

**2.6 에러 메시지 위치**
- **문제**: 에러 메시지가 입력 필드 아래에만 표시됨
- **개선 방안**: 중요한 에러는 상단 토스트로도 표시

### 3. 접근성 개선 사항

#### 🟡 중요 (개선 권장)

**3.1 이미지 대체 텍스트**
- **문제**: SVG 아이콘에 `aria-label`이 있으나 일부 누락
- **개선 방안**: 모든 아이콘 버튼에 `aria-label` 추가

**3.2 폼 라벨 연결**
- **문제**: 일부 라벨이 `for` 속성 없이 사용됨
- **개선 방안**: 모든 라벨에 `for` 속성 추가

**3.3 언어 설정**
- **현재**: `<html lang="ko">` ✅
- **추가 개선**: 다국어 지원 시 동적 변경

## 🎯 구체적 개선 제안

### 1. 에러 메시지 개선

**현재**:
```css
.input-error-message {
  font-size: 0.875rem; /* 14px */
  color: var(--color-error);
}
```

**개선**:
```css
.input-error-message {
  font-size: 0.9375rem; /* 15px */
  font-weight: 600; /* 굵게 */
  color: var(--color-error);
  display: flex;
  align-items: center;
  gap: 0.25rem;
}
```

### 2. 키보드 네비게이션 개선

**현재**:
```html
<button class="btn-icon" onclick="closeModal()">
  <svg>...</svg>
</button>
```

**개선**:
```html
<button 
  class="btn-icon" 
  onclick="closeModal()"
  tabindex="0"
  aria-label="모달 닫기"
  onkeydown="if(event.key==='Enter'||event.key===' ') closeModal()"
>
  <svg aria-hidden="true">...</svg>
</button>
```

### 3. 모바일 테이블 개선

**현재**:
```css
.glass-card {
  overflow: hidden;
}
```

**개선**:
```css
.glass-card {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch; /* iOS 부드러운 스크롤 */
}

.data-table {
  min-width: 800px;
}

@media (max-width: 768px) {
  .data-table {
    font-size: 0.875rem;
  }
}
```

### 4. 로딩 상태 개선

**추가**:
```html
<div id="loadingState" class="hidden">
  <div class="flex items-center justify-center py-8">
    <div class="animate-spin w-8 h-8 border-4 border-red-600 border-t-transparent rounded-full"></div>
    <span class="ml-3 text-gray-600">로딩 중...</span>
  </div>
</div>
```
